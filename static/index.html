SSE-Broker接口文档

1. 获取Token

1.1. 接口描述

  • 请求地址：/token
  • 请求方式: POST/GET
  • 返回数据类型：application/json
  • 用途：根据参数生成JWToken，用于SSE连接鉴权

1.2. 请求参数

1.2.1. Header 参数(POST时必须)

参数名 必选 类型/参数值 说明
Content-Type 是 application/json POST JSON

1.2.2. Body 参数(POST时必须)

参数名 必选 类型 限制条件 说明
uid 是 string 用户ID
device_id 否 string 客户端设备ID
ttl 否 int 单位秒，默认3600秒 生成的token在ttl秒后过期

1.3. 请求示例 


        POST /token HTTP/1.1
        {
            "uid": "115",
            "device_id": "x1987"
        }
        //或者
        GET /token?uid=1556

1.4. 返回示例 

{
    "code": 1,  // 1：成功，其他：失败, 全部接口均返回此格式
    "msg": "",  // 错误信息
    "result": "jwtToken",    // 生成的JWT Token, 该token的有效期基于ttl参数
    "micros": 189       // 接口耗时，单位微秒
    }

2. 发送消息

2.1. 接口描述

  • 请求地址：/send
  • 请求方式: GET/POST
  • 返回数据类型：application/json
  • 向指定uid/deviceId/全部连接的设备,发送事件或消息

2.2. 请求参数

2.2.1. Header(POST时必须)

参数名 必选 类型/参数值 说明
Content-Type 是 application/json POST JSON

2.2.2. Body 参数

参数名 必选 类型 限制条件 说明
data 是 String 消息内容
event 否 String 事件名称
uid 否 String 用户ID，多个uid以英文逗号分隔
device_id 否 String 设备ID，多个设备ID以英文逗号分隔

2.3. 请求示例 


        POST /send HTTP/1.1
        {
            "uid": "115",
            "data": "测试"
        }
        //或者
        GET /send?uid=1556&data=abc

2.4. 返回示例 

{
    "code": 1,  // 1：成功，其他：失败, 全部接口均返回此格式
    "msg": "",  // 错误信息
    "result": 3,    // 发送成功的设备数
    "micros": 189       // 接口耗时，单位微秒
    }

3. SSE连接

3.1. 接口描述

  • 请求地址：/events
  • 请求方式: GET
  • 用途：无参，客户端连接SSE服务器的地址，也是唯一需要暴露到外网的接口

3.2. 请求参数

2.2.1. Header

参数名 必选 类型/参数值 说明

3.2.2. Body 参数

参数名 必选 类型 限制条件 说明
token 是 String SSE的授权token
device 是 String 当前设备ID，不可重复

3.3. 请求示例 


        GET /events?token=abcdefg&device=xuyd
